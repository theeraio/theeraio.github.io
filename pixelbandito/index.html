Comment-based roadmap

2016-07-08T14:32:19Z

I hate having to go to more than one place for things. I'd like to try using structured comments in conjunction with a barebones github repo to replace a roadmap / ticketing system.

You'd start an empty repo, add a README, and leave comments that describe desired functionality, priority, responsible individuals, dependencies, etc. Then you'd run a utility that scans the repo and compiles your todo list.

Github issues already does a pretty good job with feature requests, but issues are still a proprietary Github-ism, right? This would be 100% portable.

P.S. For javascript, I think the decorators pattern could work instead of comments, but one of the challenges is finding a syntax that you could use to create issues and requests in any file in the repo. I think a structured comment is a better fit, since every programming language I've used has some kind of block comment feature.
